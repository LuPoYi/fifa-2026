# Middle Play(不會雙輸對沖)工作流

使用者喜歡的下注結構:兩注的贏面**聯集蓋滿所有結果**(雙輸不可能),重疊區 = 雙贏爆賺。
偏好**窄重疊、雙贏大勝**型;單邊結果小虧或保本即可。娛樂性質、小額,誠實提醒 EV ≈ 打平(vig)。

## 核心數學(必背)

- **對等對沖注碼**:已持有 A(本金 S、賠率 oA),補 B 時 → **U = S × oA ÷ oB**(兩個單邊結果損益拉平)
- **無風險門檻**:`1/oA + 1/oB < 1` → 補後三種結果全 ≥ 0,重疊區純 bonus
  - 例:oA=2.27 → oB ≥ 1.79 即無風險
- 損益:雙贏 = S(oA−1) + U(oB−1);只中A = S(oA−1) − U;只中B = U(oB−1) − S

## 常用結構(驗證「聯集蓋滿」再下)

| 結構 | 兩腿 | 雙贏區 | 蓋滿原因 |
|---|---|---|---|
| 大小 middle | Over N + Under M(N<M) | 總分在 (N,M) | 總分不可能同時 <N 又 >M |
| 讓分 middle | A −x.5 + B +y.5(y>x) | 淨勝分在中間 | 同上 |
| 比分連動(盤中) | Over 2.5 + 落後方不贏 | 有球但落後方沒贏 | 落後方要贏必灌球 |
| 雙不輸 | A不輸 + B不輸 | 平手 | 雙輸 ⟺ 兩隊都贏(不可能) |

⚠️ 名詞:SX 的「A 不輸」= 網站的 **Not (對手)**。

## 工作流

1. **賽前**:用 `python3 sx.py odds <eventId> --all` 抓全盤口,依使用者看法挑結構,列
   「雙贏/單邊A/單邊B」損益表(格式:結果|例子比分|淨損益,標明 🔒雙輸不可能)。
2. **Legging(分腿)**:使用者常先單押 value 邊,等盤中另一腿賠率變好再補。
   給 cheat sheet:各賠率水位 → 補多少(U = S×oA÷oB)→ 三格損益。
3. **背景監看**:寫輪詢腳本(參考 `watch_under*.py` pattern,scratchpad 內)用
   `run_in_background` 跑,**觸發即 break 讓主程序收到通知**。監看事件:
   - 補腿賠率 ≥ 目標門檻(報:賠率、可吃量、建議注碼)
   - 賠率掉回(窗口關閉)
   - 每次進球(比分變 → 重算劇本)
   - 總分越線(某腿已死/已中)、終場(結算)
   - 輪詢間隔 ≥6s(`/orders` 限速 20/10s);log 稀疏(2-4 分鐘一行)
   - ⚠ 終場判定:live-scores 的 `currentPeriod` 實測回 **"Full time"**(不是 "Finished"),
     判斷用 `per in ("Finished","Full time")`;periodTime 5400 = 90 分鐘打完
4. **警報時**:直接給「下多少 + 三格損益表」,提醒以使用者畫面當下價為準、看可吃量。

## 實戰教訓(2026-07-03 Swiss vs Algeria)

- 窗口開關很快:進球 → 補腿賠率跳高(1.20→1.55)→ 幾分鐘沒球就掉回(→1.35)。
  **到合理價就動,別貪無風險門檻**(等 1.79 常等到窗口關閉)。
- 第二顆球會重開窗口;比分每變一次,「還差幾球」的劇本就要重算。
- 盤中書單薄:報價時一併報「最佳價可吃量」,不足額要說。

## 誠實提醒(每次都講)

結構只決定賠付形狀,不會把負 EV 變正;這是娛樂/控波動玩法。
詳細理論見使用者個人 HackMD「運動博弈不會雙輸對沖玩法」。
